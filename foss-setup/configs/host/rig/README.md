# rig host-level config — systemd unit index (glue-11 / glue-13)

This directory mirrors the rig's (CachyOS `192.168.10.12`) **host-level** config that
is *not* managed by an ansible role. `site.yml` only converges `base` / `docker` /
`tailscale` / `backup` / `state` — it has **zero** `fstab` / mount / host-unit tasks — so
the files here are **documentation-only**: nothing re-applies them, and a live edit that
skips the repo (or vice-versa) is silent drift until it bites. Land every change in both.

glue-13 audited **every** custom unit under the rig's `/etc/systemd/system/` and pinned each
to a single canonical source. The rule (same as the task pitfall): a unit already stored
somewhere in this repo is **not** re-copied here — a second identical copy is a new intra-repo
drift axis. Units owned by a **separate** repo get a pointer, never a duplicate. This table is
the map.

## External-repo units (pointers, not copies)

Two rig units are owned by the separate **`local-ai-tooling`** repo
(`github.com/btabaska/local-ai-tooling`, checked out on the rig at
`~/Documents/GitHub/local-ai-tooling/`). Mirroring them here would create cross-repo drift, so
this repo only points at where they live:

- **`fleet-mcp.service`** — read-only fleet-inspection MCP server (ai-01). The unit's own header
  says `Install: sudo cp ops/fleet-mcp.service /etc/systemd/system/`; `ExecStart` runs
  `ops/fleet_mcp.py` out of the `local-ai-tooling` checkout. Live is byte-identical to
  `local-ai-tooling/ops/fleet-mcp.service`.
- **`ollama.service.d/override.conf`** — Ollama tuning drop-in (`OLLAMA_HOST=0.0.0.0`, KV-cache
  `q8_0`, `KEEP_ALIVE=0`, …). Emitted by `local-ai-tooling/scripts/01-ollama-tune.sh`
  (`tee $OVERRIDE_FILE`); that script is the source of truth. Regression-guarded by the
  `rig-ollama-keepalive` verification check.

## Drift coverage

The statically-mirrored foss-setup units are byte-compared against live on the mini + rig by the
`unit-file-drift` verification check (`scripts/verification/unit-drift-check.sh`, run from the
mini runner). The ansible-managed backup units are **templated** and converge daily via
ansible-pull, so they self-heal rather than being byte-checked; the external units are the
owning repo's drift axis, not this one's.
