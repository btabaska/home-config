1The alerting path — why no pings, and the fix chain

Your question: "ntfy loads to a blank page, says no subscriptions — how will I know if I'm getting phone pings?" Answer: right now, you wouldn't be. Here's the whole picture.

The blank page itself is expected — ntfy is set to deny anonymous access, and web subscriptions live per-browser, so a fresh browser always looks empty. But behind that, the delivery path to your phone is broken in three stacked ways, and three other monitoring tools generate nothing at all. Fix them in this order:

2Your questions, answered

Is AdGuard running the best blocklists?

No — both instances run only the single stock "AdGuard DNS filter" (good baseline, ~155k rules, auto-updating). The architecture underneath is textbook (primary → local recursive Unbound with DNSSEC; secondary on an independent Quad9 path for resilience). Recommendation for "it just works, don't break sites": add HaGeZi Multi Normal + OISD Big on both — both engineered for near-zero false positives. Skip the aggressive Pro++/1Hosts tiers. Second win: the two AdGuards drift independently today (parity is luck); bakito/adguardhome-sync keeps blocklists/rewrites identical automatically while preserving the NAS's separate upstream. Minor cleanup: a dead :853 DoT listener on the mini, and 90-day query logs worth trimming to 7.

Do my *arr apps have TRaSH rules? — Already done. ✓

Yes, fully. Recyclarr has been quietly syncing the complete TRaSH profiles every Sunday — Sonarr on WEB-1080p (37 custom formats, all 171 series), Radarr on HD Bluray+WEB (40 formats, all 316 movies), TRaSH-correct naming everywhere. The "no container running" is because it's a one-shot cron job by design. Nothing to implement; only trivial hygiene (delete 6 unused factory profiles, tidy two Lidarr artists).

Can Sunshine wake the rig? — Yes, on LAN, already. ✓ (with a gap)

Moonlight already wakes the rig natively — every client (iPhone, iPad, Apple TV, Steam Deck, desktop) has a "Wake PC" action that sends the magic packet; ~30s boot and you're in. The gap: over Tailscale (away from home) WoL can't traverse — that needs a LAN-side waker. Recommended: an ntfy topic wake-rig with a tiny subscriber on the mini (using the wake-rig.sh that already exists and is proven). Then an iOS Shortcut is a one-tap wake from anywhere. (On iOS, check the Moonlight app has Local Network permission — a known silent-fail.)

Can we add a WoL button to home.tabaska.us / wake-on-click?

Homepage will never have a native WoL button (maintainers rejected it — a page-load poller would wake the rig every time you open the dashboard). Two real paths:

  1. Now (simple): a "Wake rig" tile on Homepage whose link hits the ntfy wake-rig topic from #above — works from any device, LAN or remote.
  2. The dream ("click ai.tabaska.us and it just wakes"): the caddy-wol module. Key discovery — your Caddy is already a custom build (for the Cloudflare DNS plugin), so adding this is one line. Clicking a rig service while it sleeps sends the wake and the page simply finishes loading (~30-60s) as the service comes up. One thing to verify first: that the broadcast actually escapes Caddy's container network.

Recommended: ntfy waker now (covers phone + remote), caddy-wol as phase-2 polish. Alternatively UpSnap — a proper WoL dashboard — does double duty as the friend-facing wake button for game servers (§4).
