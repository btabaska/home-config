How to use this guide

This is a self-contained tracker for migrating off the Apple/iOS ecosystem onto your own hardware. Finish the Backbone tier first, then pick parallel workstreams below ‚Äî they are independent queues, not one linear list. Tasks are ordered within each track by logical progression; SYNC tasks block their track until done, Async tasks can run anytime. Click a workstream header to collapse or expand its task list; completed workstreams auto-collapse. Check a task to mark it done; progress is saved in this browser automatically. Click any task for steps, commands, config files, and a concrete "done when" check. Delegating to an AI agent? Start with the Agent handoff prep tier above the backbone ‚Äî then filter AI handoff only for rollout tasks the agent can run.

AI handoff ‚Äî agent can run via SSH/sudo with minimal input AI + you ‚Äî agent does bulk work; you provide a key or spot-check Synchronous ‚Äî blocking within its track (a dependency or one-way door) Async ‚Äî run unattended / in parallel / anytime Host the task runs on Required ‚Äî foundational infrastructure only; use Required only to see the ~21-task baseline before feature workstreams

Infrastructure baseline (Required only): operator MacBook prep ‚Üí UniFi backup + VLANs + zone firewall ‚Üí Tailscale + SSH to every host ‚Üí NAS volume layout + SMB/NFS + local snapshots + Container Manager ‚Üí Docker on the Mac mini. That unlocks every parallel workstream. If backbone is mostly done, finish nas-01 (snapshots) and glue-01 (UPS) first ‚Äî they gate Immich, Paperless, and off-site backup.

Cheap local sensor shopping list

Grounded picks for the Zigbee backbone (Smart home track). All pair with Zigbee2MQTT/ZHA, need no vendor hub, and run fully local. Street prices mid-2026 (USD); links go to the manufacturers. Buy the Sonoff SNZB-P line for best price/reliability, Aqara as the premium option, Third Reality for plugs and the siren leak sensor.

Start here: you need exactly one coordinator. Then mix and match sensors. Every mains-powered device (smart plug) is also a Zigbee router ‚Äî spread 5-7 around the house for a stable mesh, pair devices near the coordinator first, then move them. Avoid Aqara-branded hubs if you mix brands.

Pinholes carved through the blocks: IoT ‚Üí gateway for DNS (53) and DHCP (67/68) is always allowed; IoT ‚Üí gateway admin (HTTP/HTTPS/SSH) is blocked; IoT ‚Üí RFC1918 (10/8 ¬∑ 172.16/12 ¬∑ 192.168/16) is blocked. *Hue can be internet-blocked once it's fully local in HA; Nest still needs Google's cloud.

DNS path

filtered + encrypted

Client ‚Üí AdGuard Home (ad/tracker block) ‚Üí Unbound (recursive + DNSSEC) ‚Üí root servers. A firewall NAT rule redirects all outbound :53 to the resolver and blocks known DoH endpoints so nothing bypasses the filter. AdGuard on the NAS is the DHCP secondary.

Remote access

no open inbound ports

Tailscale mesh (WireGuard, 41641/udp out) links every host + phones + the off-site seedbox over 100.64.0.0/10 with MagicDNS. Public web apps use Caddy HTTPS at *.tabaska.us (auto-cert, optional Cloudflare DNS-01). The only deliberately forwarded ports are specific game servers (or a Playit.gg tunnel).

Off-network
Seedbox "Betty" (clean datacenter IP, Deluge-only P2P) and cloud backups (Backblaze B2 ¬∑ Hetzner ¬∑ Synology C2) sit outside the LAN ‚Äî the NAS *arr stack imports finished downloads over an rclone SFTP mount, so the ISP never sees a swarm and off-site copies survive a fire/theft.

Runs off your network (non-local)

Deliberately not on your hardware ‚Äî so your ISP never sees a swarm and your off-site copies survive a fire/theft.

Seedbox "Betty" ‚Äî Bytesized "Stream +3"

off-site ¬∑ 3000 GB ¬∑ 6-10 TB/mo upload cap ¬∑ ‚Ç¨16/mo (~$18)

Deluge + slskd. Betty runs torrent (Deluge) and Soulseek (slskd) P2P off-site. The NAS *arr stack searches indexers and sends torrent grabs to Deluge; Soularr on the NAS drives remote slskd for Soulseek. All imports use the rclone SFTP mount at /volume1/mounts/seedbox-files. No *arr apps, Soularr, or beets on Betty.
