How this works — staged autonomous runs

Plan v3 (2026-07-07): all remaining work is organized into eight Runs. The AI executes each run autonomously over SSH and the repo, then hands back a chat summary of what it did, what's verified, and exactly what it needs from you — the run's human-gate list (credentials, GUI clicks, purchases, pairings). You work the gate list, say "go", and the next run starts. Each run header below shows its goal, live progress, the AI-vs-you task split, and the open gates. The three tasks the 2026-07-07 audit proved regressed are flagged REOPENED with the reason.

Use the Runs ⇄ Tracks toggle to switch between the run plan (default) and the classic workstream view. SYNC tasks block their batch; Async can run anytime. Check a task to mark it done — progress saves in this browser, merges from the committed docs/progress.json when served over HTTP, and exports/imports as JSON either way.

AI — fully autonomous over SSH/repo AI+vault — autonomous, using credentials from the local secrets vault Mixed — AI does the bulk; you do a short step (pairing, approval, test) You — GUI-only, physical, decision, or purchase Gate — the human input that unblocks the task REOPENED — marked done before, disproven by the 2026-07-07 audit

Infrastructure baseline (Required only): operator prep → UniFi VLANs + zone firewall → Tailscale + SSH to every host → NAS volume layout + snapshots → Docker on the Mac mini → convergence repairs (fix-06/07/14). The Required only filter shows the current baseline set, computed live from the task data.

Cheap local sensor shopping list

Buy the Sonoff SNZB-P line for best price/reliability, Aqara as the premium option, Third Reality for plugs and the siren leak sensor.

Zone-Based Firewall policy

Stateful — return traffic is automatic, so only the initiating direction is shown. Rows = source zone, columns = destination. Specific allows sit above the broad blocks (UniFi custom policies outrank built-ins; use Reorder).

From ↓ / To →TrustedIoTCamerasWorkGuestInternet
Trusted—ALLOWALLOWblockblockALLOW
IoTblock—blockblockblockALLOW*
Camerasblockblock—blockblockblock
Workblockblockblock—blockALLOW
Guestblockblockblockblock—ALLOW
Pinholes carved through the blocks: IoT → gateway for DNS (53) and DHCP (67/68) is always allowed; IoT → gateway admin (HTTP/HTTPS/SSH) is blocked; IoT → RFC1918 (10/8 · 172.16/12 · 192.168/16) is blocked. *Hue can be internet-blocked once it's fully local in HA; Nest still needs Google's cloud.

DNS path

filtered + encrypted

Client DHCP DNS chain (fail-open): #1 mini AdGuard → #2 NAS AdGuard → #3 gateway. Mini path: AdGuard → Unbound (recursive + DNSSEC). NAS secondary uses independent DoT upstream. NAT :53 redirect + DoH blocking (dns-05) only after the three-tier chain is verified — see configs/network/dns-resilience-plan.md.

Remote access

no open inbound ports

Tailscale mesh (WireGuard, 41641/udp out) links every host + phones + the off-site seedbox over 100.64.0.0/10 with MagicDNS. Public web apps use Caddy HTTPS at *.tabaska.us (auto-cert, optional Cloudflare DNS-01). The only deliberately forwarded ports are specific game servers (or a Playit.gg tunnel).

Off-network
Seedbox "Betty" (clean datacenter IP, Deluge-only P2P) and cloud backups (Backblaze B2 · Hetzner · Synology C2) sit outside the LAN — the NAS *arr stack imports finished downloads over an rclone SFTP mount, so the ISP never sees a swarm and off-site copies survive a fire/theft.

Runs off your network (non-local)

Deliberately not on your hardware — so your ISP never sees a swarm and your off-site copies survive a fire/theft.

Seedbox "Betty" — Bytesized "Stream +3"

off-site · 3000 GB · 6-10 TB/mo upload cap · €16/mo (~$18)

Deluge + slskd. Betty runs torrent (Deluge) and Soulseek (slskd) P2P off-site. The NAS *arr stack searches indexers and sends torrent grabs to Deluge; Soularr on the NAS drives remote slskd for Soulseek. All imports use the rclone SFTP mount at /volume1/mounts/seedbox-files. No *arr apps, Soularr, or beets on Betty.

Deluge (download + seed ONLY) Tailscale (userspace)
